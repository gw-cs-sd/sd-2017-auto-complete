Our Project

We built an autocomplete system that is able to handle large amounts of data while still remaining fast, resouce-efficient, and accurate. We used terabytes of ngram data from the Google Books corpus, moving it through a pipeline that included pre-processing the data, storing and indexing it, querying it, and making it available via a web API.

I personally worked on the data pre-processing, developing a process to clean, condense, and sort terabytes of data without overwhelming available processing power or memory, on the web API, making our completions easily available to any system that can make web requests and handle JSON, and on sample client applications.
